# 점수 업로드 엑셀 표준 양식 다운로드 — 설계

- 날짜: 2026-06-13
- 대상: 교수(관리자) 대시보드 → 점수 일괄 업로드 기능
- 상태: 승인됨 (구현 계획 작성 예정)

## 1. 배경 / 목적

교수가 중간·기말·출석 점수를 엑셀로 일괄 업로드할 때, 어떤 컬럼/형식으로
파일을 만들어야 하는지 매번 직접 맞춰야 한다. 업로드 모달에서 **현재 재학생이
미리 채워진 표준 엑셀 양식을 내려받아**, 점수만 입력해 그대로 다시 올릴 수 있게 한다.

## 2. 확정된 요구사항 (브레인스토밍 결정)

- **양식 내용**: 현재 등록된 재학생 전원의 `학번`·`이름`을 미리 채우고, 점수
  칸은 비운다. (`이름`은 교수 식별용 참고 컬럼 — 업로드 파서는 무시)
- **적용 범위**: 점수 3종(중간·기말·출석) 업로드 모달에만 추가. 신규 일괄
  등록 모달에는 추가하지 않는다.
- **점수 칸**: 이미 점수가 있는 학생도 빈칸으로 둔다(덮어쓰기 위험 회피, 선택 사양).

## 3. 접근 (채택: A)

현재 `package.json`에 이미 있는 `xlsx`(SheetJS)로 **클라이언트에서 워크북을
생성**하고 다운로드를 트리거한다. 학생 목록은 `AdminPage`가 이미 메모리에
들고 있는 `students` 상태를 모달에 prop으로 넘겨 재사용한다 (추가 네트워크 호출/
의존성 없음).

- 기각 B: 모달이 다운로드 시점에 Supabase를 직접 재조회 → 중복 쿼리·결합도 증가.
- 기각 C: `/public`에 정적 양식 파일 → 재학생 자동 채움 불가(요구사항과 충돌).

## 4. 컴포넌트 / 인터페이스

### 4.1 신규 모듈 `src/lib/excelTemplate.ts`
관심사 분리: 데이터 가공(순수) 과 다운로드(부수효과)를 나눈다.

```ts
import type { ScoreField, Student } from './supabase'

// 순수 함수: 헤더 + 학생 행을 2차원 배열로 반환 (단위 테스트 가능)
// 반환 예: [['학번','이름','기말'], ['20240001','홍길동',''], ...]
export function buildScoreTemplateRows(
  field: ScoreField,
  students: Pick<Student, 'student_number' | 'name'>[],
): string[][]

// 부수효과: 워크북 생성 후 XLSX.writeFile 로 브라우저 다운로드 트리거
// 파일명 예: '기말_점수_양식.xlsx'
export function downloadScoreTemplate(
  field: ScoreField,
  students: Pick<Student, 'student_number' | 'name'>[],
): void
```

- 점수열 헤더 = `SCORE_LABEL[field]` (`중간`/`기말`/`출석`)
- 컬럼 순서: `학번`, `이름`, `<점수 라벨>`
- 시트명: 점수 라벨, 파일명: `${SCORE_LABEL[field]}_점수_양식.xlsx`

### 4.2 `src/components/ExcelUploadModal.tsx`
- props에 `students: Pick<Student,'student_number'|'name'>[]` 추가.
- `mode.kind === 'score'` 이고 파일 미선택 상태(`totalRows === 0`)일 때, 드롭존
  하단에 **"표준 양식 다운로드 (.xlsx)"** 버튼 렌더 → `downloadScoreTemplate(mode.field, students)` 호출.
- register 모드에서는 버튼을 렌더하지 않는다.

### 4.3 `src/pages/AdminPage.tsx`
- `<ExcelUploadModal ... students={students} />` 로 전체 학생 목록 전달
  (검색 필터된 `filtered`가 아니라 원본 `students`).

## 5. 데이터 흐름 / 라운드트립

1. 교수가 점수 업로드 모달 열기 → "표준 양식 다운로드" 클릭.
2. `downloadScoreTemplate`가 `students`로 워크북 생성 → `.xlsx` 다운로드.
3. 교수가 점수열만 채워 같은 파일을 업로드.
4. 기존 점수 파서가 `학번`과 점수 라벨(`기말` 등) 열을 읽음. `이름` 열은
   매핑 대상이 아니므로 자동 무시 → **라운드트립 정상 동작**.

## 6. 엣지 케이스

- 재학생 0명: 헤더만 있는 양식이 다운로드된다(정상).
- 점수 미입력 행: 재업로드 시 기존 동작대로 "점수가 비어있습니다"로 분류되어
  건너뛴다 → 기존 점수를 덮어쓰지 않는다(안전). 단, 결과 화면에 '실패'로
  표시되는 기존 UX는 그대로 유지한다.

## 7. 검증

이 프로젝트는 테스트 인프라가 없다. 따라서:
- `tsc --noEmit` 타입체크 통과.
- 점수 3종 각각 양식 다운로드 → 파일 내용(헤더/학번/이름/빈 점수열) 확인.
- 다운로드한 양식에 점수 입력 후 재업로드 → 정상 갱신(라운드트립) 확인.

테스트 프레임워크 추가는 범위 외(기존에 테스트가 전무). 필요 시 별도 작업으로
`buildScoreTemplateRows` 순수 함수에 대한 단위 테스트를 추가할 수 있다.

## 8. 범위 외 (YAGNI)

- 점수 칸에 기존 점수 미리 채우기.
- 테스트 프레임워크 도입.
